# Customize the golf course starter

This site keeps course-specific content in `src/data/` and `src/content/` wherever practical. Start with the files below, then run `npm run check` and `npm run build` after edits.

Before editing, collect the client’s answers in [CLIENT_ONBOARDING.md](./CLIENT_ONBOARDING.md). After editing, complete [DEMO_CONTENT_CHECKLIST.md](./DEMO_CONTENT_CHECKLIST.md).

## Course identity, location, and contact details

Edit `src/data/business.ts` to change:

- Course and short names
- Tagline and business description
- Production URL
- Phone, email, and address
- Map coordinates and directions link
- Booking URL
- Public hours
- Social and Google Business Profile links
- Price range

The header, footer, contact page, schema, social card, and many calls to action read from this file.

## Colors and visual theme

The working Tailwind design tokens live in `src/styles/global.css` inside `@theme`. Update those values to change the rendered site. Keep `src/data/theme.ts` synchronized; it serves as a simple reference for tools or future theme controls. Also update `themeColor` in `src/data/siteSettings.ts` if the primary green changes.

## Navigation and site settings

- Edit `src/data/navigation.ts` for primary and footer navigation.
- Edit `src/data/siteSettings.ts` for the mobile labels, course announcement, default social image, theme color, robots default, and footer utility links.
- The course status announcement is static. Update it manually when conditions change or disable it with `enabled: false`.

## Homepage and local search copy

- `src/data/home.ts` controls the homepage hero, quick cards, outings, clubhouse feature, stats heading, and final CTA.
- `src/data/localSeo.ts` controls the local SEO section.
- `src/data/pageContent.ts` contains contact, guide CTA, privacy, and accessibility copy.

## Golf content and pricing

- `src/data/course.ts`: course stats, tees, rating and slope, all 18 holes, playing tips, policies, and the tee-selection note.
- `src/data/rates.ts`: walking/riding prices, notes, and rate disclaimer.
- `src/data/menu.ts`: food and drink sections, items, prices, specials, and disclaimer.
- `src/data/memberships.ts`: tiers, prices, benefits, FAQs, and disclaimer.
- `src/data/events.ts`: event types and planning steps.
- `src/data/activities.ts`: range, lessons, junior golf, simulators, clinics, and pro shop cards.

Every displayed rate and price is sample content. Confirm it before launch.

## Guides

Add a Markdown file under `src/content/guides/`:

```md
---
title: A useful guide title
description: A concise search and social description.
published: 2026-06-22
eyebrow: Local golf guide
---
```

The guide index and detail page are generated automatically. Use internal links such as `/course`, `/tee-times`, and `/contact` where they help the reader.

## Images and placeholders

The Course page uses `VisualPlaceholder.astro` instead of large demo images. Put optimized WebP or AVIF photos in `public/images/`, replace each placeholder with an Astro `<Image>` or HTML `<picture>`, and provide useful alt text when the image conveys information. Decorative images should use an empty alt attribute.

The social card at `/social-card.svg` is generated by `src/pages/social-card.svg.ts` from `business.ts`. Change the colors or layout in that endpoint if needed; course name, location, and tagline update automatically.

## SEO and production domain

- Edit per-page titles and descriptions in `src/data/seo.ts`.
- Replace the demo URL in `astro.config.mjs`, `src/data/business.ts`, and `public/robots.txt`.
- Confirm the generated social card and canonical links after building.
- Add the real Google Business Profile URL to `business.social.googleBusiness` when available.

## Forms

The contact and event forms intentionally do not submit. Labels, required fields, and helper content are reusable through `FormField.astro` and `src/data/forms.ts`. Connect a form service, update the privacy policy, remove the disabled state, and test spam protection before accepting data. If forms are unnecessary, remove their page sections.

## Cloudflare Pages

Use build command `npm run build` and output directory `dist`. `public/_headers` adds conservative security headers without a CSP. Once all map, booking, analytics, image, and form providers are known, consider adding a tested Content Security Policy.
